Transport Layer

The Transport Layer enables communications between application processes on different hosts. It provides essential services like multiplexing and demultiplexing, reliable data transfer and flow control.

Multiplexing and demultiplexing are essential functions to allow multiple devices to share the network. Multiplexing allows multiple applications to share the same transport layer protocol, whilst demultiplexing ensures that received data is directed to the correct application.

The two main protocols used are TCP, which ensures reliable and ordered delivery, and UDP, which offers a faster but less reliable connectionless service.

Reliable data transfer ensures that data is delivered error-free and in the correct order. This is done by using sequence numbers to number each byte of data, which allows the receiver to reassemble the data in the correct order. Flow control is used to prevent the sender from overwhelming the receiver with data. This is done by the receiver sending messages to the sender to tell it to stop sending data.

TCP operates as a connection-oriented transport protocol. It establishes a connection using a 3 way handshake and insures data integrity through acknowledgments.

Congestion control in TCP prevents excessive data transmission that can lead to network congestion. AIMD (Additive Increase, Multiplicative Decrease) gradually increases the sending rate until packet loss occurs, then reduces it. Slow Start allows TCP to start with a low sending rate and increase it to prevent congestion. Retransmit & Recovery quickly detects and retransmits lost packets by using duplicated acknowledgments.
